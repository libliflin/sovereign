# Kind Cluster Management

## Cluster Identity

- **Name:** `sovereign-test`
- **Context:** `kind-sovereign-test`
- **Config:** `cluster/kind/kind-config.yaml` (3 nodes: 1 control-plane + 2 workers)
- **Values:** `cluster-values.yaml` (root of repo, generated by bootstrap)

Always pass `--context kind-sovereign-test` or `--kube-context kind-sovereign-test` to kubectl/helm.

## Creating the Cluster

If the snapshot says "cluster does not exist":

```bash
# Check first
kind get clusters 2>/dev/null | grep -q "^sovereign-test$" && echo "exists" || echo "missing"

# Create (idempotent — bootstrap.sh checks first)
./cluster/kind/bootstrap.sh --cluster-name sovereign-test --output cluster-values.yaml

# Install foundations (Cilium, cert-manager, sealed-secrets, MinIO)
./cluster/kind/install-foundations.sh --cluster-name sovereign-test
```

Wait for nodes to be Ready after creation:
```bash
kubectl wait --for=condition=Ready nodes --all --context kind-sovereign-test --timeout=120s
```

## Checking Cluster Health

```bash
# Nodes
kubectl get nodes --context kind-sovereign-test

# All pods
kubectl get pods -A --context kind-sovereign-test

# Foundation health (Cilium must be running for anything else to work)
kubectl get pods -n kube-system -l k8s-app=cilium --context kind-sovereign-test
kubectl get pods -n cert-manager --context kind-sovereign-test
kubectl get pods -n sealed-secrets --context kind-sovereign-test
kubectl get pods -n minio --context kind-sovereign-test
```

## Resetting the Cluster

Nuclear option — only when cluster is corrupted beyond repair:

```bash
kind delete cluster --name sovereign-test
./cluster/kind/bootstrap.sh --cluster-name sovereign-test --output cluster-values.yaml
./cluster/kind/install-foundations.sh --cluster-name sovereign-test
```

## Kind-Specific Incompatibilities

These components **cannot run in kind** — disable them, don't waste cycles:

| Component | Issue | Resolution |
|-----------|-------|------------|
| **Falco** | eBPF driver-loader needs kernel headers absent in linuxkit | Disable: `--set falco.enabled=false` |
| **Rook-Ceph** | Needs raw block devices, kind has none | Use `standard` or `local-path` StorageClass |
| **ceph-block StorageClass** | Doesn't exist in kind | Change PVC storageClassName to `standard` |

## StorageClasses Available

```
standard          rancher.io/local-path   (kind default)
local-path        rancher.io/local-path   (created by install-foundations.sh)
```

There is NO `ceph-block`. Any chart requesting `ceph-block` will have PVCs stuck in Pending.

## Docker Desktop DNS Workaround

Kind nodes use Docker's bridge DNS (192.168.65.254), which has no record for `harbor.sovereign.local`.
When Harbor is deployed, inject its ClusterIP into kind node /etc/hosts:

```bash
HARBOR_IP=$(kubectl get svc harbor -n harbor --context kind-sovereign-test -o jsonpath='{.spec.clusterIP}' 2>/dev/null)
if [[ -n "$HARBOR_IP" ]]; then
    for node in $(kind get nodes --name sovereign-test); do
        docker exec "$node" bash -c "grep -q 'harbor.sovereign.local' /etc/hosts || echo '$HARBOR_IP harbor.sovereign.local' >> /etc/hosts"
    done
fi
```

## Loading Images into Kind

Kind cannot pull from localhost registries. Use `kind load`:

```bash
# From a local docker image
kind load docker-image <image:tag> --name sovereign-test

# From a tar archive
kind load image-archive <path.tar> --name sovereign-test
```
