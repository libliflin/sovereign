## Operating Principles (permanent — do not overwrite)

- **STOP THE LINE**: When any gate fails, fix it before touching anything else.
  The gate output is ground truth. Your belief about what should work is irrelevant.
  ceremonies.sh writes the exact failure (helm lint output, shellcheck line, etc.)
  into sprint._lastSmokeTestFailures / _lastProofOfWorkFailures. ralph.sh injects
  these into your prompt. You will see exactly what broke. Act on it immediately.

- **Never build on red**: A broken helm lint in charts/cilium blocks ALL chart work,
  not just cilium. Fix it first. One defect per stop.

- **Gate output is injected automatically**: You do not need to re-run smoke tests
  manually to discover failures. If ceremonies.sh caught something, the exact error
  is already in your prompt under "## ⚠ FAILURE CONTEXT". Read it before writing code.

## Codebase Patterns
(Updated by Ralph after each iteration — check here first for conventions discovered)

- Domain variable: always {{ .Values.global.domain }} in Helm templates
- Storage class: always {{ .Values.global.storageClass }} — default is "ceph-block"
- All ingresses use TLS, cert from cert-manager
- Keycloak OIDC client IDs follow pattern: sovereign-<service>
- ArgoCD apps live in argocd-apps/<tier>/<service>-app.yaml
- Namespace naming: one namespace per service, named after the service
